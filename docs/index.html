Дизайн-система в коде: как ей пользоваться

Единый источник цветов, типографики, отступов и компонентов. Всё, что тут описано, живёт в токенах и синхронизировано с Figma — фронтенду не нужно подбирать hex, px или веса шрифтов вручную.

Основы (цвета, типографика, отступы, радиусы) — это точные значения токенов. Превью компонентов ниже — схематичные иллюстрации на токенах; эталонный вид, размеры и все состояния смотрите в Figma.

Обзор

Система построена на трёх уровнях токенов. Правило простое: цвет — только из семантических токенов, размеры — из примитивов, текст — из стилей Geist. Тогда light/dark и общий визуальный язык получаются «бесплатно».

Уровень 1

Примитивы

Сырые значения: color.*, space.*, radius.*, font.size.*. Напрямую в компонентах для цвета не используются.

Уровень 2

Семантика

Смысловые алиасы: background/*, text/*, border/*, interactive/*, status/*. Их и используем.

Уровень 3

Типографика

Текст-стили Geist (Display, Heading/*, Body/*, Label/*, Strong/*). Один стиль на каждый кусок текста.

01

Подключение токенов

Токены собираются из tokens/ в CSS-переменные и типизированную TS-карту. Импортируешь один раз в корне приложения — дальше ссылаешься на переменные.

Один импорт в корне

// src/main.tsx
import "../build/css/variables.light.css";  // :root — примитивы + светлая семантика
import "../build/css/variables.dark.css";   // [data-theme="dark"] — тёмные оверрайды
import "../build/css/typography.css";       // классы .text-* + --font-family-base

Цвет и размеры — через CSS-переменные

.card {
  background: var(--background-secondary);
  color:      var(--text-primary);
  border:     1px solid var(--border-default);
  border-radius: var(--radius-lg);
  padding:    var(--space-4);      /* 16px */
  gap:        var(--space-2);      /* 8px  */
}

Текст — класс или типизированная карта

// вариант A — CSS-класс из typography.css
<h3 className="text-heading-h3">Заголовок карточки</h3>
<p  className="text-body-small">Подпись</p>

// вариант B — типизированная карта для инлайновых стилей
import { Typography } from "../build/ts/typography";
<h3 style={Typography.HeadingH3}>Заголовок</h3>
Тема переключается одним атрибутом. Поставь data-theme="dark" на <html> — все цвета перевернутся сами, потому что имя переменной в обеих темах одинаковое. Второй цвет для тёмной темы вручную задавать нельзя.

07

Действия

Кнопки — единственные компоненты, уже реализованные в React (src/components/). Остальные пока живут в Figma как спецификация — собирай их по тем же токенам, следуя паттерну Button.tsx.

Превью компонентов на этой странице — схематичные. Они собраны вручную на токенах системы (цвета и типографика — настоящие), но это не точный рендер компонентов из Figma и не готовый код. Пропорции, скругления, тени и отдельные состояния могут отличаться. Эталон вида, размеров и всех состояний — в Figma; что реализовано в коде (React), а что только в Figma (Figma) — показывают метки и «Полный каталог».

Варианты: primary · secondary · ghost · glass · primary-dark и статусные success · warning · error. Размеры sm · md · lg, флаг fullWidth, слоты leadingIcon/trailingIcon.

14

Как не сломать систему

✓ Цвет — только семантика

var(--background-*), --text-*, --border-*, --interactive-*, --status-*. Никаких hex и --color-* примитивов в компоненте.

✓ Размеры — только шкала

--space-*, --radius-*, --font-size-*, --icon-size-*. Значение «между» не изобретаем — округляем к ближайшему.

✓ Текст — только стиль Geist

Класс .text-* или Typography.*. Не задаём font-size/weight руками.

✓ Light & dark — бесплатно

Компонент обязан работать в обеих темах без единой лишней строки — за это отвечают токены.

✓ Нет токена — добавь его

Не хватает значения — добавь в tokens/ и отзеркаль в Figma. Сырой литерал допустим лишь для нетокенизируемого (например прозрачность стекла).

✓ Новый компонент — по паттерну

Один файл в src/components/, типизированные пропсы, именованный экспорт. Ориентир — Button.tsx.

Источник правды — Figma. Токены и текст-стили синхронизированы в tokens/ и собираются в build/. Эта страница — снимок системы для фронтенд-команды.
