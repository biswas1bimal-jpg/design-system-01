Дизайн-система на переменных

03

Одна переменная — путь до GitHub

На примере interactive/primary: как переменная Figma становится строкой в токенах, потом CSS-переменной, и всё это лежит в репозитории.

① Figma Variable interactive/primary Light → blue/600
Dark → blue/500
② tokens/semantic.json "interactive": { "primary": { "value": "{color.blue.600}" } } алиас на примитив
③ build/css --interactive-primary: var(--color-blue-600); + dark override в [data-theme="dark"]
④ GitHub background: var(--interactive-primary) потребляется в src/
04

Где это лежит в GitHub

Переменные Figma синхронизированы в tokens/, собираются в build/ и запушены в ветку. CI пересобирает build/ автоматически при изменении токенов.

biswas1bimal-jpg/design-system-01 ветка · claude/figma-button-components-3gdvz3
tokens/ # переменные Figma в коде (source) ├─ primitives.json # color.* / space.* / radius.* / font.size.* / weight.* ├─ semantic.json # light + dark: background/text/border/interactive/status ├─ typography.json # текст-стили Geist └─ icon.json # icon.size.* build/ # сгенерировано автоматически (не редактируется руками) ├─ css/variables.light.css # :root — примитивы + светлая семантика ├─ css/variables.dark.css # [data-theme="dark"] — тёмные оверрайды ├─ css/typography.css # классы .text-* └─ ts/tokens.ts # типизированная карта токенов .github/workflows/ └─ build-tokens.yml # CI: пересобирает build/ при изменении tokens/
Переменные Figma → tokens/ → build/ → src/. Один источник правды, одно значение управляет дизайном и кодом одновременно.
